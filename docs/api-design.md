# 多模态反诈智能助手 — API 设计

> **核心依据**：按赛题四大模块 + 任务清单逐条对应  
> ML 团队提供统一多模态分析服务，后端通过内部 HTTP 调用

---

## 1. 用户与角色管理模块 (端：网页端)

> 对应赛题：**个性化风险评估**（角色属性影响风险阈值）、**行为画像**（长短期记忆）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/user/register` | 用户注册 (网页端) |
| POST | `/api/user/login` | 登录，返回 Token |
| GET | `/api/user/profile` | 获取用户信息、角色画像、已绑定监护人列表 |
| PUT | `/api/user/profile` | 更新角色属性（年龄段、职业、风险偏好） |
| PUT | `/api/user/config` | 设置风险灵敏度与干预策略（动态调整风险阈值） |
| GET | `/api/user/memory` | 获取行为画像（供 ML 个性化风险评估） |

---

## 2. 监护人联动模块 (端：微信小程序)

> 对应赛题：**监护人联动**（高风险时自动通知）、**多重预警机制**、**误报率<5%**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/guardian/bind` | 绑定监护人（姓名、电话、关系） |
| DELETE | `/api/guardian/unbind` | 解绑监护人 |
| GET | `/api/guardian/whitelist` | 获取信任联系人/账号白名单（降低误报） |
| POST | `/api/guardian/whitelist` | 签加信任联系人/账号到白名单 |
| PUT | `/api/guardian/notify-policy` | 设置通知偏好（实时推送 vs 周期汇总） |
| GET | `/api/guardian/summary` | 监护人主动拉取被监护人的安全周报 |
| POST | `/api/guardian/intervention/{wardId}` | **远程干预**：监护人向被监护人下发干预指令，后端实时推送至对方 App |

---

## 3. 多轮对话模块

> 对应赛题：**任务清单第4条**（对话历史存储与检索，多轮上下文一致性）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/chat/send` | **核心接口**：发送消息（文本+附件），返回 AI 回复 + 风险判定结果 |
| GET | `/api/chat/history` | 获取历史对话记录（保证上下文一致性） |

---

## 4. 多模态检测模块

> 对应赛题：**多模态输入**（文本/音频/视觉）、**智能识别引擎**、**任务清单第6条**（批量评估）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/detect/multimodal` | 直接提交多模态数据进行诈骗检测（非对话场景） |
| POST | `/api/detect/batch` | 批量检测（测评集场景，返回 F1-score / 准确率） |

---

## 5. 预警与干预模块

> 对应赛题：**分级预警**、**监护人联动**、**实时干预**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/alert/history` | 获取历史预警记录 |
| GET | `/api/alert/detail/{id}` | 获取单次预警的判定依据 |
| **WS** | `/ws/alert` | 实时推送通道：接收系统预警 + 监护人远程干预指令 |

---

## 6. 安全监测报告模块

> 对应赛题：**报告生成**（可视化安全监测报告，含风险说明与防御建议）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/report/generate` | 生成指定时间段的安全监测报告 |
| GET | `/api/report/list` | 获取历史报告列表 |

---

## 7. 知识库自适应进化模块

> 对应赛题：**自适应进化模块**（新案例清洗后导入向量数据库）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/knowledge/upload` | 上传新诈骗案例，自动触发清洗与向量化入库 |

---

## 内部调用：ML 分析服务

| 方法 | 地址 | 说明 |
|------|------|------|
| POST | `http://ml-service/api/predict` | 发送多模态数据，返回风险分值、诈骗类型、置信度 |
